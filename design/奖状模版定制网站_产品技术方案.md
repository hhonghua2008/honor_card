# 奖状模板定制网站 · 产品技术方案

> 版本：v1（规划稿） · 状态：待评审 · 作者：bluey
> 范围：纯前端 Web 应用，桌面优先，MVP 不做后端/账号

---

## 0. 方案结论（TL;DR）

- **形态**：纯前端 SPA，桌面优先，无需登录、无后端。
- **技术栈**：React + TypeScript + Vite；画布编辑器用 **Fabric.js**；状态用 Zustand；样式用 TailwindCSS。
- **核心能力**：模板画廊（20+ 模板，含照片奖状专区）、在线编辑器（文字 + 照片裁剪遮罩）、本地保存、客户端生成分享链接、下载 PNG/PDF。
- **分享链接**：把"项目状态"压缩编码进 URL（pako + base64），纯前端实现，无需后端；照片过大时采用"占位重传"策略。
- **MVP 不做**：账号体系、云端存储、批量生成（留作 P2/P3，其中批量生成教师/HR 场景强需求，建议尽早排期）。

## 1. 产品定位与用户场景

### 1.1 定位
面向"需要快速做一张好看奖状"的非设计人群，提供 **选模板 → 改内容 → 出图** 的极简闭环。关键词：免费、即开即用、打印级清晰度、照片奖状一键遮罩。

### 1.2 目标用户与场景

| 用户 | 典型场景 | 核心诉求 |
| --- | --- | --- |
| 教师 / 班主任 | 期末、运动会、比赛颁奖 | 照片奖状、校园风模板（批量生成在 P2） |
| HR / 行政 | 员工表彰、周年庆 | 企业风、品牌色、可加司徽 |
| 活动 / 社群组织者 | 比赛、志愿者、亲子活动 | 活动风、可加 logo |
| 家长 | 给孩子做成长奖状 | 童趣、照片奖状 |
| 电商 / 微商 | 荣誉证书、代理授权 | 品牌模板、可署名 |

### 1.3 差异化价值
- 比 Word 模板好看、比 Canva 简单、比找人设计快。
- **照片奖状一键遮罩**（圆形 / 圆角 / 心形 / 相框）解决最大痛点。

## 2. 用户流程

```
浏览模板 → 选择模板 → 在线编辑 → 预览 → 保存 / 分享 / 下载
```

- **在线编辑** 内含：文字（标题/姓名/事由/日期/落款）、照片（上传→遮罩裁剪）、背景（换色/上传）、元素（形状/图标/印章/签名）。
- **照片奖状**：编辑阶段上传照片 → 按模板遮罩形状自动裁剪 → 与文字一同导出。
- **分享链接**：保存阶段将项目状态编码进 URL，纯前端实现。

## 3. 功能设计

### 3.1 模板画廊
- **分类**：校园表扬、竞赛获奖、员工表彰、活动荣誉、节日/励志、照片奖状（专区）、空白自定义。
- **交互**：卡片展示缩略图/名称/标签；支持搜索、筛选、排序（热门/最新）。
- **配色**：每个模板内置 2–4 套主题预设（一键换色）。

### 3.2 编辑器（核心，WYSIWYG）
一切皆为"图层对象"，画布所见即所得。

**图层类型**

| 类型 | 说明 |
| --- | --- |
| background | 纯色 / 渐变 / 图片 / SVG 底图 |
| text | 可编辑文字（标题、姓名、事由、日期、落款） |
| photo | 照片占位（含遮罩形状 + 可选装饰边框），用户上传替换 |
| decoration | SVG 形状 / 飘带 / 印章 / 边框 / 图标 |

**编辑能力**
- **文字**：双击内联编辑；字体（思源黑体 / 宋体 / 楷体 / 艺术字）、字号、颜色、粗斜体、对齐、字间距、旋转、拖动缩放。
- **照片**：上传 → 自动适配占位（cover/contain）→ 在占位内拖动/缩放/旋转 → 应用遮罩（圆/圆角矩形/心形/相框）→ 替换/移除。人脸优先裁剪（P3，ML）可选。
- **背景**：换色、上传、从素材库选。
- **元素**：加形状、图标、印章、签名（上传或输入）。
- **通用**：撤销/重做、对齐辅助线、画布缩放。

### 3.3 照片奖状专项
- 模板内置 `photo` 占位，定义：位置、尺寸、宽高比、**遮罩形状**、装饰边框。
- 用户上传后按占位遮罩裁剪，可微调构图。
- 导出时照片随模板一起渲染为位图。

### 3.4 保存与导出
- **保存项目**：用 `localForage`（IndexedDB）存"项目 JSON"（模板 id + 图层修改 + 用户照片 dataURL），支持多项目、命名、删除、继续编辑。
- **下载**：
  - **PNG**：高分辨率（按模板 DPI，如 A4@300dpi = 2480×3508）。
  - **PDF**：用 jsPDF 生成打印级 A4 PDF。
  - **JPG**：社交分享用。
- **分享链接（纯前端）**：
  - 项目状态 → JSON → pako 压缩 → base64 → 写入 URL hash（`#p=...`）。
  - 打开链接解码还原为只读预览，可"另存为我的项目"继续编辑。
  - **照片处理**：dataURL 可能过长，方案：① 仅编码文字/布局（照片用占位，接收方重传）；② 限制照片尺寸/质量后内联。MVP 采用①②结合（小图内联、大图占位提示重传）。
  - **上限**：URL 约 64KB 安全；超出则提示"作品过大，暂不支持链接分享，可直接下载"。

### 3.5 响应式
- **桌面优先**：左图层/工具 + 中画布 + 右属性面板。
- **平板/手机**：属性面板改底部抽屉，画布缩放适配；MVP 以桌面为主，移动端保证可浏览/下载。

## 4. 技术架构（纯前端）

### 4.1 架构概述
- 纯前端 SPA，无后端；模板与素材走静态 CDN（或同域 `public/`）。
- 编辑器基于 **Fabric.js** 管理画布对象与导出。
- 状态：**Zustand** 管理项目/编辑器状态。
- 路由：模板画廊 / 编辑器 / 我的项目 / 分享查看。
- 导出：Fabric `toDataURL`(PNG) + jsPDF(PDF)。
- 分享：URL hash 编码。

### 4.2 目录结构（草案）

```
honor_card/
  src/
    pages/          # Gallery, Editor, MyProjects, ShareView
    editor/         # canvas, layers, toolbar, property-panel
    templates/      # 模板 scene JSON + 缩略图
    lib/            # export(png/pdf), share(encode/decode), storage
    assets/         # 字体、装饰 SVG
  public/templates/ # 模板静态资源（底图/缩略图）
```

### 4.3 模板数据模型

```ts
type Template = {
  id: string
  name: string
  category: string
  tags: string[]
  thumbnail: string
  canvas: { width: number; height: number; dpi: number; orientation: 'portrait' | 'landscape' }
  themePresets: ThemePreset[]          // 配色预设
  layers: Layer[]                      // 初始场景
}

type Layer =
  | { type: 'background'; fill?: string; src?: string }
  | { type: 'text'; text: string; fontFamily: string; fontSize: number; fill: string; left: number; top: number; ... }
  | { type: 'photo'; mask: 'circle' | 'rounded' | 'rect' | 'heart' | 'frame'; frameSrc?: string; left: number; top: number; width: number; height: number }
  | { type: 'decoration'; src: string; left: number; top: number; ... }
```

### 4.4 项目数据模型

```ts
type Project = {
  id: string
  name: string
  templateId: string
  layers: Layer[]                       // 用户修改后的完整图层
  photoData?: Record<string, string>   // 占位id -> dataURL（可选内联分享）
  updatedAt: number
}
```

### 4.5 关键技术选型与理由

| 选型 | 理由 |
| --- | --- |
| Fabric.js | 成熟画布引擎，WYSIWYG、拖拽缩放、遮罩、`toDataURL` 导出一站到位 |
| React + TS + Vite | 组件化、类型安全、构建快 |
| Zustand | 轻量状态，适合编辑器高频更新 |
| TailwindCSS | 快速搭 UI |
| jsPDF + Fabric | 客户端生成打印级 PDF |
| pako + base64 | 分享链接压缩编码 |
| localForage (IndexedDB) | 存大体积项目（含照片）比 localStorage 稳 |

## 5. 模板生产流程
- 设计同学用 Figma 设计 → 导出底图(SVG/PNG) + 标注文字/照片占位坐标 → 工程转成 Template JSON。
- 提供"模板校验脚本"：检查坐标 / DPI / 字体是否齐全。
- MVP 先手工沉淀 20+ 模板（含 5+ 照片奖状），后续可做"模板管理后台"(P3)。

## 6. 路线图

| 阶段 | 内容 | 说明 |
| --- | --- | --- |
| **P1（MVP）** | 画廊 + 编辑器 + 照片 + 本地保存 + 下载 + 分享链接 | 纯前端，验证需求 |
| **P2** | 批量生成（CSV 导入名单→批量出图打包）、更多模板、字体库扩展 | 教师/HR 高频 |
| **P3** | 模板管理后台、人脸裁剪、更多装饰素材、用户上传模板 | 平台化 |
| **P4** | 移动端深度适配 / 微信小程序、账号云同步 | 传播与留存 |

## 7. 风险与对策

| 风险 | 对策 |
| --- | --- |
| 分享链接超长（含照片） | 大图占位重传；超出提示下载 |
| 字体版权 | 用开源/自有字体（思源系列），商用需授权提示 |
| 导出清晰度 | 按 300dpi 渲染，避免 CSS 缩放失真 |
| 模板侵权 | 模板原创或 CC 授权，注明来源 |
| 移动端编辑体验 | MVP 桌面优先，移动端先保证浏览/下载 |
